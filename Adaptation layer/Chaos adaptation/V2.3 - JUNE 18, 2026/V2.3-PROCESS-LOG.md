### A2. `app/Ai/Adaptation/RuntimeNarratorTemplateBuilder.php`

**What:** Three changes: cap reduction, pre-render anchor guard, post-render token guard.

**Change 1 — cap reduction (line ~30):**
```php

// AFTER:

```
Rationale: D8 v2 requires a 65k cap. The old 128k allowed bloated prompts that caused voice drift from over-dense Section 12.

**Change 2 — new `assertAnchorFieldsPresent()` method** (called at top of `build()`):
```php
private function assertAnchorFieldsPresent(array $voiceProfile, Story $story, SessionAdaptation $session): void
{
    $missing = [];
    if (empty($voiceProfile['voice_anchor']))       { $missing[] = 'voice_anchor'; }
    if (empty($voiceProfile['anchor_card']))        { $missing[] = 'anchor_card'; }
    if (empty($voiceProfile['runtime_self_check'])) { $missing[] = 'runtime_self_check'; }

    if ($missing !== []) {
        throw new \RuntimeException(sprintf(
            'Runtime narrator assembly requires a V2.3 voice_profile with voice_anchor, anchor_card, and runtime_self_check. '
            . 'Missing fields: %s (story %d, session %d). Re-run adaptation for this story.',
            implode(', ', $missing), $story->id, $session->session_number,
        ));
    }
}
```
This is called before rendering — so if the voice profile is a pre-V2.3 profile, we fail immediately with a clear message rather than producing a degraded prompt.

**Change 3 — new `assertRenderedPromptClean()` method** (called after render, before returning):
```php
private function assertRenderedPromptClean(string $rendered, Story $story, SessionAdaptation $session): void
{
    if (preg_match('/\\{\\{[^}]+\\}\\}/', $rendered)) {
        throw new \RuntimeException(sprintf(
            'Assembled runtime narrator prompt contains unmapped {{…}} tokens (story %d, session %d). '
            . 'Template slot wiring is incomplete.',
            $story->id, $session->session_number,
        ));
    }
    if (! str_contains($rendered, '[OPENING_SCENE_INJECTION_POINT]')) {
        throw new \RuntimeException(sprintf(
            'Assembled runtime narrator prompt is missing [OPENING_SCENE_INJECTION_POINT] (story %d, session %d). '
            . 'Section 13 must contain the injection point token.',
            $story->id, $session->session_number,
        ));
    }
}
```
Unmapped `{{...}}` means a slot variable name in the template couldn't be resolved (wiring gap). Missing injection point means Section 13 was accidentally removed during template edits.

**`dropVoiceQuotes()` docblock update:**
Added explicit comment that this method `NEVER touches the top-level anchor fields (voice_anchor, anchor_card, runtime_self_check)` — these are load-bearing and survive every compression pass intact.

**Compression cascade order in `build()` method:**
```
compression_attempts: full → compressed_source → titles_only_source
voice_compression: full → drop_quotes
```
Source events are compressed first. Voice material (specifically the heavy quote arrays in `author_voice_dna_profile`) is only stripped as a last resort before the editorial split exception. `voice_anchor` exemplar prose is NEVER in the compression path — it is at the top level and `dropVoiceQuotes()` only touches `author_voice_dna_profile`.

public static function dnaBansAndAnchor(array $voiceProfile): array
{
    if (empty($voiceProfile['voice_anchor'])) {
        throw new \RuntimeException(
            'dnaBansAndAnchor() requires a V2.3 voice_profile with voice_anchor populated. '
            . 'Re-run Voice Lock for this story before running Phase 3.'

    // voice_anchor prose is load-bearing — include verbatim, never stripped.
    $base['voice_anchor'] = $voiceProfile['voice_anchor'];
    $base['anchor_card']  = $voiceProfile['anchor_card'];

    return $base;
}

public static function dnaBansAndAnchorCard(array $voiceProfile): array

## ARCHITECTURAL DECISIONS MADE

### Decision 1: `[OPENING_SCENE_INJECTION_POINT]` not `{{PHASE_3_COLD_OPEN_PROSE}}`

The D8 v2 deliverable shows `{{PHASE_3_COLD_OPEN_PROSE}}` at the Section 13 assembly slot. We substituted it with `[OPENING_SCENE_INJECTION_POINT]` — the existing token that `ChaosEngineService` injects at session start. This is correct: the cold open prose is NOT baked at adaptation time. It is injected LIVE from `entry_point_diagnosis.cold_open` when `isSessionStart=true`. Baking it at assembly would make the prompt stale after any Phase 3 re-run.

### Decision 2: `first_choice_spec` stored in `entry_point_diagnosis` JSON

The spec is returned by `EntryPointDiagnosisAgent` as a top-level field and stored alongside `cold_open`, `editorial_diagnosis`, etc. in `session_adaptations.entry_point_diagnosis`. This means `ChoiceDesignJob` reads it as `$session->entry_point_diagnosis['first_choice_spec']`. No schema migration required — `entry_point_diagnosis` is a JSON column.

### Decision 3: `dnaBansAndAnchorCard()` in Phase 5, not `dnaBansAndAnchor()`

Voice Anchor exemplar prose (6–8 × 90–150 words = ~700–1200 additional words) would inflate the Phase 5 prompt beyond its budget. Phase 5 writes player-facing authored prose; for that job, the Anchor Card binary rules are sufficient. The full exemplars are only needed by the runtime narrator (D8) which directly imitates them. This is the explicit separation the plan required.

### Decision 4: `dnaBansAndAnchor()` / `dnaBansAndAnchorCard()` throw, not degrade

Both methods throw `RuntimeException` if the required anchor fields are missing. This is intentional — a pre-V2.3 profile cannot silently produce a Phase 3 D10 prompt or a Phase 5 D4-patched prompt that obeys the Anchor Card, because those fields don't exist. The error is explicit: "Re-run Voice Lock for this story before running Phase 3/5." No fallback to old behavior.

### Decision 5: `fourteen_point_audit_protocol` preserved as legacy, not removed

Old profiles have this field. Removing it from the schema would cause parse failures on old profiles. We kept the field definition but removed `->required()` and added a legacy note. New V2.3 profiles use `build_time_qa_protocol` instead. Old profiles keep working until deliberately re-adapted.

### Decision 6: `voice_decay_prevention_protocol` preserved, not rendered

The VDPP is preserved in the schema (screenwiter only) for backward compat. It is NOT referenced in the D8 v2 template — the new voice decay mechanism is entirely handled by `voice_anchor[]` (imitation), `anchor_card[]` (re-read every turn), and `runtime_self_check[]` (pre-delivery pass). VDPP was the old counting-based approach; the new approach is imitation + binary/local rules.

### Decision 7: EntryPointDiagnosisJob throws BEFORE status update

```php
// V2.3: Phase 3 (D10) requires voice_anchor and anchor_card.
// dnaBansAndAnchor() throws RuntimeException if either is absent.
$voiceProfile = VoiceProfilePromptSlice::dnaBansAndAnchor((array) ($adaptation->voice_profile ?? []));

try {
    $session->update(['session_status' => SessionAdaptationStatusEnum::ENTRY_POINT_DIAGNOSIS]);
    ...
```

The slice call is outside the `try` block. If it throws, the session status is NOT updated. This prevents the session from appearing to be "in progress" when it cannot proceed.

## Post-implementation fix — ChaosEngineService sec13Anchor regression

**Discovered:** During cross-file contract review after sprint completion.

**Bug:** Sprint A replaced the old V2.2 runtime template's section headings (`=== SECTION N — ... ===` format) with the D8 v2 format (`### SECTION N: ... [...]`). The `ChaosEngineService::renderSystemPrompt()` continuation path uses `strpos($prompt, $sec13Anchor)` to locate Section 13 and mechanically strip it before the narrator sees it on mid-session turns. The anchor string was hardcoded as `'=== SECTION 13 —'` and was not updated to match the new heading format.

**Impact:** On every continuation turn (`isSessionStart=false`), the section strip condition `($sec13Start !== false && ...)` would always fail (since `strpos` returns `false`). Section 13's instructions and the literal `[OPENING_SCENE_INJECTION_POINT]` token would remain in the system prompt as unprocessed text. The narrator would see the raw token mid-session.

**Fix:** `ChaosEngineService.php` line ~232: `$sec13Anchor = '=== SECTION 13 —'` → `$sec13Anchor = '### SECTION 13:'`.

**Files changed:**
- `app/Services/ChaosEngineService.php`

**No re-adaptation required.** Fix is runtime-only — applies to all cached `runtime_narrator_prompt` strings on the next turn.
