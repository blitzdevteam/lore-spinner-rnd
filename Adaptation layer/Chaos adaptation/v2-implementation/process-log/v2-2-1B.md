## Phase 1A — `VoiceLockSchema.php`

**`isScreenwriter()` — defensive format routing:**

```php
// Before
return strtoupper((string) $detectedFormat) === 'SCREENPLAY';

// After
return in_array(strtoupper((string) $detectedFormat), [
    'SCREENPLAY', 'TELEPLAY', 'PILOT', 'LIMITED_SERIES',
], true);
```

FormatDetectionAgent currently only emits `SCREENPLAY | NOVEL` — the expansion future-proofs routing when subtypes arrive without changing the detection agent or pipeline order.

**New `$includeScreenwriterFields` block additions:**

All new schema objects added inside the existing `if ($includeScreenwriterFields)` conditional. Novelist schema is not touched.

1. `$metricSpecSchema` — shared object type: `target`, `floor`, `ceiling`, `confidence`, `sample_size`. Used by all metrics in `numerical_enforcement_layer`.

2. `numerical_enforcement_layer` (Section M) — sub-objects:
   - `punctuation`: 10 metric specs (period_density_per_100w, comma_density_per_100w, semicolons, exclamation_marks_narration, em_dashes, question_marks_narration, question_marks_dialogue, ellipses_narration, ellipses_dialogue, period_to_comma_ratio)
   - `rhythm`: 11 metric specs (sentence_length_1_3w through sentence_length_26_plus_w — 7 buckets, fragment_rate, verb_first_percentage, ing_opening_percentage, rhythm_change_frequency)
   - `dialogue_ceilings_per_character`: array of objects with avg/p90/p95/max/speech_count/confidence
   - `opener_distribution`: array of per-type specs
   - `word_length`: average_chars + 4 bucket percentages

3. `rhythm_transition_architecture` (Section N) — `transition_matrix` is a 4×4 object keyed by `ultra_short / short / medium / long`, each row having the same four keys. Also: `rhythm_change_frequency`, `max_consecutive_same_category`, `signature_moves[]`, `anti_patterns[]`.

4. `beat_architecture_protocol` (Section O) — `beat_frequency`, `beat_vocabulary` (4 sub-groups), `beat_placement`, `beat_density_by_context`.

5. `scene_transition_compression_protocol` (Section P) — `closing_line_avg_length`, `closing_line_type_distribution` (5 keys), `closing_line_examples[]`, `transition_guidance`.

6. `screenplay_to_prose_protocol` — replaced the previous flat array with an object:
   - `element_rules[]`: `{ screenplay_element, prose_translation_rule }` — same item shape as the old array
   - `quantitative_translation_mappings[]`: `{ screenplay_metric, source_value, prose_target, drift_ceiling, rationale }` — minimum 6 per deliverable table

**Legacy read shim (runtime/slices):** If a pre-v2 profile has `screenplay_to_prose_protocol` as a bare array, consumers treat it as `element_rules`. New extractions always write the object shape.

7. `voice_decay_prevention_protocol` — added to `$topLevel` (the return array), outside `author_voice_dna_profile`, inside `if ($includeScreenwriterFields)`:
   - `re_anchoring_trigger` (string)
   - `passage_level_enforcement_checks[]` (array of strings)
   - `drift_detection_metrics[]` (array of strings)

## Phase 1B — `VoiceLockChapterAgent.php`

Added a required `metric_counts` object to the screenwriter chapter fragment schema. All fields are **producer-neutral** — today the chapter LLM fills them; a future deterministic metric extractor can populate the same fields without changing merge schema, `voice_profile` schema, or runtime rendering.

Fields added inside the existing `if (VoiceLockSchema::isScreenwriter($this->detectedFormat))` block:

**`metric_counts` object** — 19 required raw-count fields:

| Field | Purpose |
|-------|---------|
| `action_line_count` | Denominator for all action-line percentage metrics |
| `action_line_word_count` | Total words in action lines |
| `dialogue_speech_count` | Total speeches (denominator for dialogue metrics) |
| `dialogue_word_count` | Total words in dialogue |
| `punctuation_counts` | Object: period, comma, semicolon, exclamation, em_dash, question, ellipsis |
| `fragment_line_count` | Action lines ≤5 words |
| `verb_first_line_count` | Action lines opening with verb |
| `ing_opening_line_count` | Action lines opening with -ing participle |
| `line_length_bucket_counts` | Object: `1_3w`, `4_5w`, `6_8w`, `9_12w`, `13_18w`, `19_25w`, `26_plus_w` |
| `opener_type_counts` | Object: article, pronoun, character_name, verb, negation, preposition, ing, all_caps |
| `word_length_bucket_counts` | Object: chars_1_3, chars_4_5, chars_6_8, chars_9_plus |
| `beat_count` | 1–2 word standalone beats |
| `scene_closing_line_count` | Closing lines before scene breaks |
| `scene_closing_word_count` | Word count of closing lines |
| `scene_closing_type_counts` | Object: image, action, status, dialogue_adjacent, beat |
| `rhythm_transition_matrix_counts` | 4×4 grid: after `ultra_short/short/medium/long` → next-category counts (all 16 cells) |
| `first_action_line_bucket` | Category of first action line in chapter (for inter-chapter boundary stitching) |
| `last_action_line_bucket` | Category of last action line in chapter |
| `dialogue_speech_lengths_by_character` | Array: `{ character, speech_lengths_w[], speech_count, max_speech_length_w }` |

**Qualitative evidence fields added:**
- `beat_candidates[]`: `{ beat_text, placement_context, function }`
- `scene_closing_samples[]`: `{ closing_lines[], scene_context }`
- `confidence_sample_size_notes`: string — notes for sparse metrics in this chapter

**Existing qualitative fields retained:** `action_line_samples`, `action_line_metrics_note`, `dialogue_metrics_note`

## Phase 1C — Chunk Metric Aggregation Contract

Implemented in prompts (Phase 2). The contract:

1. Chapter fragments return **raw counts** (no percentages).
2. Merge **sums all counts** across chunks first.
3. Merge derives percentages, targets, floors, ceilings, confidence tiers from summed totals.
4. Never average percentages across chunks.
5. Zero-occurrence ABSOLUTE bans: only when summed count is zero across all chunks and the denominator covers the full source.
6. Rhythm transition matrix: merge adds inter-chapter boundary transitions using `last_action_line_bucket` → `first_action_line_bucket` of consecutive chunks.
7. Dialogue P90/P95/MAX: merge concatenates `speech_lengths_w[]` per character across all chunks, then calculates from the combined list.
